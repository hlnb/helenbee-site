# HelenBurgess Post Builder

Use this skill for post-level work in the HelenBurgess.id.au repo.

## Use this skill when

- creating a new post
- expanding a rough draft into a publishable article
- restructuring an existing post for flow and readability
- standardising front matter and metadata
- improving intros, conclusions, headings, excerpts, or descriptions
- preserving voice while tightening prose

## Do not use this skill for

- broad site redesigns
- large template refactors
- routing or collection architecture changes
- unrelated technical debugging unless directly tied to a post workflow

## Working principles

- preserve the author’s voice
- keep first-person writing natural
- improve clarity without sanding away the humanity
- respect subtlety and emotional nuance
- do not flatten lived experience into generic wellness content

## Default post workflow

1. inspect the existing post or source notes
2. identify the piece type:
   - reflective essay
   - journal-derived article
   - personal/philosophical commentary
   - craft/technology/life crossover piece
3. preserve or complete front matter using repo conventions
4. improve structure, readability, and flow
5. add or refine:
   - title
   - excerpt or summary
   - description
   - tags/categories
   - headings if appropriate
6. validate that the post still sounds like HelenBurgess

## Editorial rules

- explain clearly, but do not over-explain feelings
- prefer grounded language
- avoid false uplift and tidy clichés
- keep openings strong and specific
- keep conclusions honest and earned
- preserve continuity with the site’s themes of healing, resilience, creativity, Stoicism, mindfulness, and lived experience

## Metadata guidance

When creating or revising a post:

- ensure front matter is complete and valid
- preserve slugs unless explicitly asked to change them
- preserve categories/tags unless there is a strong reason to adjust them
- keep SEO support helpful but secondary to voice

## Quality checks

Before finishing:

- check front matter formatting
- check heading structure
- check readability and paragraph flow
- check that the post still sounds like the author
- note any assumptions or missing metadata
